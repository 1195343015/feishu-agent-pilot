# 二、项目结果展示

## 1、Demo 展示

### 核心演示流程

**场景一：Web 端全流程演示**

1. 打开 `localhost:5173`，页面顶部栏显示 AI 模型健康状态、WebSocket 连接状态、在线协作人数
2. 在「对话指令」面板输入自然语言指令，例如："帮我生成一份QQ音乐的需求文档和6页PPT"
3. Agent 自动执行任务编排：
   - **理解意图**：LLM 解析用户指令，提取关键信息
   - **生成文档**：调用智谱 glm-4.5-air 模型生成结构化 Markdown 需求文档，实时写入「协同文档」面板
   - **生成 PPT**：基于文档内容二次调用 LLM 生成演示稿大纲，在「演示稿」面板展示
   - **任务步骤**：左侧「任务编排」面板实时显示每个步骤的执行状态
4. 用户可在「协同文档」中手动编辑内容，支持插入表格、图片等富媒体块
5. 点击「生成交付物」，系统自动：
   - 创建真实飞书文档并写入内容
   - 生成标准 PPT 文件（pptxgenjs）并上传飞书云空间
   - 返回可点击的飞书文档链接和 PPT 链接
6. 点击「下载文档」/「下载 PPT」可直接下载本地文件

**场景二：飞书机器人对话演示**

1. 在飞书群聊或单聊中 @Agent-Pilot 机器人，发送："帮我写一个穿越火线的需求文档和PPT"
2. 机器人第一轮回复：「📄 文档已生成：https://feishu.cn/docx/xxx」+ "正在生成 PPT 演示稿，请稍候..."
3. 机器人第二轮回复：「✅ PPT 已生成：https://feishu.cn/file/xxx」
4. 用户直接点击链接即可在飞书内查看和编辑文档和 PPT
5. 发送"进度"可查询当前任务状态

**场景三：多端实时协同演示**

1. 同时打开两个浏览器窗口（或手机端），连接同一个 workspace
2. 一端编辑文档内容，另一端实时同步显示变化
3. 顶部栏实时显示在线人数，多用户 presence 感知
4. 断网后本地仍可编辑（IndexedDB 持久化），恢复网络后自动无冲突合并

**场景四：等待体验优化**

1. Agent 执行期间，任务卡片下方自动提示"AI 生成中，来局 2048？"
2. 点击打开内置 2048 小游戏，游戏内顶部显示 AI 任务实时状态
3. AI 完成后状态条变绿，显示"查看结果"按钮，一键回到工作界面

**技术栈**：
- **前端**：React 19 + Vite + Zustand + Yjs + y-websocket + lucide-react

- **AI**：智谱 glm-4.5-air（OpenAI 兼容接口）
- **协同**：Yjs CRDT + IndexedDB 本地持久化 + WebSocket 实时同步
- **交付**：飞书开放平台 API（文档创建、云空间上传）

### 2.2 核心代码片段

**两步式 LLM 调用（文档 + PPT 分离生成）**

前端 `sendMessage` 拆分为两步串行调用，文档生成后立即可见，PPT 异步跟进：

```typescript
// 第一步：生成文档
const generation = await requestAgentGeneration(syncUrl, prompt, context);
// 文档立即可见
text.delete(0, text.length);
text.insert(0, generation.documentMarkdown);

// 第二步：基于文档生成 PPT
const slideResult = await requestSlideGeneration(syncUrl, prompt, generation.documentMarkdown);
// PPT 写入协同状态
slides.forEach(slide => currentWorkspace.slides.push([slideToYMap(createSlide(slide))]));
```

**飞书机器人两步回复**

文档和 PPT 分别生成后独立推送飞书消息，用户无需等待全部完成：

```typescript
// 第一步：文档生成完立刻回复链接
const documentId = await createFeishuDocument(title, markdown, []);
await sendFeishuText(chatId, `📄 文档已生成：${toFeishuDocLink(documentId)}\n\n正在生成 PPT...`);

// 第二步：PPT 生成完再回复
const presentationId = await createFeishuPresentation(`${title} - PPT`, slides);
await sendFeishuText(chatId, `✅ PPT 已生成：${toFeishuPresentationLink(presentationId)}`);
```

**Yjs CRDT 多端协同**

基于 Yjs 的 CRDT 算法实现无冲突实时同步，IndexedDB 提供离线持久化：

```typescript
const doc = new Y.Doc();
const workspace = getSharedWorkspace(doc);
const localPersistence = new IndexeddbPersistence(`agent-pilot:${workspaceId}`, doc);
const provider = new WebsocketProvider(syncUrl, workspaceId, doc, { connect: true });

// 监听文档变化，双向同步
workspace.document.observe(() => {
  set({ documentText: workspace.document.toString() });
});
```

**Agent 任务状态机**

完整的任务生命周期管理，支持 running / done / failed / waiting_confirm 状态流转：

```typescript
type AgentStepType = "plan" | "doc_generate" | "slide_generate" | "delivery";
type AgentStepStatus = "pending" | "running" | "done" | "failed" | "waiting_confirm";
```

## 3、项目亮点介绍

### 3.1 一键闭环，从对话到交付

用户只需一句自然语言指令，Agent-Pilot 自动完成「意图理解 → 文档生成 → PPT 生成 → 飞书交付」全链路。无需在多个应用间手动切换、复制粘贴，**将原本需要 30-60 分钟的流程压缩到 2 分钟以内**。

### 3.2 双入口、双体验

- **Web 端**：可视化仪表盘，实时展示任务步骤、文档内容、PPT 大纲、在线人数，适合精细编辑和审阅
- **飞书机器人**：零门槛对话式交互，两步推送结果链接，适合快速生成和移动端使用

两个入口共享同一套 Yjs 协同状态，Web 端生成的内容飞书端可直接查看和编辑。

### 3.3 多端实时协同

基于 Yjs CRDT 算法，支持多用户同时在线编辑：
- WebSocket 实时双向同步
- IndexedDB 离线持久化，断网可继续编辑
- 网络恢复后自动无冲突合并
- 在线人数实时感知

### 3.4 真实的飞书生态集成

不是 mock 或假链接，而是通过飞书开放平台 API 实现真实的文档创建和文件上传：
- 调用飞书 `docx/v1/documents` 接口创建在线文档
- 使用 `drive/v1/files/upload_all` 上传 PPT 到飞书云空间
- 返回可直接访问的飞书链接

### 3.5 体验细节打磨

- **等待不无聊**：Agent 运行时自动提示玩 2048，游戏内显示 AI 任务实时状态
- **渐进式反馈**：文档和 PPT 分步生成、分步推送，用户无需空等
- **主动澄清**：模糊指令时 Agent 主动询问，而非盲目执行
- **进度查询**：发送"进度"即可了解当前任务状态
- **移动端适配**：响应式布局，手机端也有良好的操作体验

## 4、AI 亮点介绍

### 4.1 AI 驱动的模块化场景编排

项目实现了赛题要求的全部 6 个核心场景，每个场景由 AI Agent 串联编排：

| 场景 | AI 作用 | 实现方式 |
|------|---------|----------|
| A. 意图入口 | 自然语言理解、意图分类 | 正则分类（澄清/进度/交付/任务）+ LLM 意图提取 |
| B. 任务规划 | 拆解子任务、生成步骤 | LLM 结构化 JSON 输出（steps + documentMarkdown） |
| C. 文档生成 | 需求文档自动撰写 | glm-4.5-air 结构化生成，支持多轮迭代 |
| D. 演示稿生成 | 基于文档提取 PPT 结构 | 二次 LLM 调用，文档 → 幻灯片映射 |
| E. 多端协同 | CRDT 冲突自动解决 | Yjs + WebSocket + IndexedDB 离线支持 |
| F. 交付归档 | 飞书文档/PPT 自动创建 | 飞书 API + pptxgenjs 本地生成 + 云端上传 |

### 4.2 两步式 LLM 生成架构

不同于传统的"一次生成所有内容"方案，我们采用了**两步式分离生成**：

1. **第一步（文档生成）**：LLM 根据用户指令和上下文生成结构化需求文档
2. **第二步（PPT 生成）**：LLM 基于已生成的文档内容，提取核心信息生成演示稿结构

这种设计的优势：
- **更快的首屏响应**：文档生成后立即可见，用户无需等 PPT 一起完成
- **更好的内容质量**：PPT 基于文档内容生成，而非凭空编造，内容一致性更高
- **更灵活的错误处理**：任一步骤失败不影响另一步骤的结果

### 4.3 人机协作分工

| 环节 | AI 负责 | 人负责 |
|------|---------|--------|
| 意图理解 | 解析自然语言、分类意图 | 输入指令（文本/语音） |
| 内容生成 | 文档撰写、PPT 结构生成 | 审阅、修改、补充细节 |
| 排版美化 | PPT 主题配色、版式布局 | 微调标题和内容 |
| 协同同步 | 冲突自动合并、状态同步 | 多端编辑、确认交付 |
| 交付归档 | 创建飞书文档、上传 PPT | 选择交付时机、分享链接 |

**核心理念**：AI 是主驾驶（Pilot），负责从理解到生成的全链路自动化；人是副驾驶（Co-pilot），在关键节点进行审阅、修正和决策。

### 4.4 模型选型思路

- **glm-4.5-air**：选择智谱轻量级模型，在中文场景下表现优秀，响应速度快（关闭推理模式后单次调用 ~1-3 秒）
- **OpenAI 兼容接口**：通过标准 `/chat/completions` 接口调用，模型可随时替换为 GPT-4、Claude 等
- **结构化 JSON 输出**：使用 `response_format: { type: "json_object" }` 确保 LLM 输出可直接解析
- **推理模式优化**：发现 glm-4.5-air 默认开启链式推理导致延迟高达 30 秒，通过 `thinking: { type: "disabled" }` 关闭后延迟降至 1-3 秒，提升 10 倍以上

### 4.5 智能等待体验

创新性地将等待时间转化为用户价值：
- Agent 运行时自动提示内置 2048 小游戏
- 游戏界面集成 AI 任务状态条，用户边玩边看到实时进度
- AI 完成后游戏内直接显示"查看结果"按钮，无缝切换回工作

### 4.6 高级 Agent 能力

- **主动澄清**：输入模糊指令时，Agent 主动询问而非盲目执行（如"你好"、"帮我"等）
- **进度查询**：发送"进度"/"到哪了"等关键词，Agent 自动汇报当前任务状态
- **多轮迭代**：基于已有文档和 PPT 内容，Agent 可根据新指令进行增量修改，而非从头生成
- **飞书意图路由**：在飞书消息中自动识别"生成文档"、"查进度"、"生成交付物"等意图，路由到不同处理流程
